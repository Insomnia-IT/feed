## Главный остаточный риск

Протокол синхронизации scanner пока принимает timestamp от клиента. Без versioned endpoint нельзя одновременно гарантировать переход старых устройств, per-kitchen server cursor и отсутствие пропусков. Рекомендуется добавить v2 с opaque cursor и держать v1 до завершения полевого upgrade window.
